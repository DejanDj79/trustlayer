Frontier Hackathon - Week 3 Video
Project: TrustLayer
Format: Screen-only demo (no face camera)
Target length: 60-75 seconds

==================================================
1) Voiceover Script (Week 3 Final, Relative to Week 2)
==================================================

Hi everyone, this is our Week 3 update for TrustLayer.

In Week 2 we focused on explainability and alert reliability.
In Week 3 we focused on transparency, product polish, and judge-ready clarity.

The biggest upgrade is score transparency.
We introduced a new Score Math section that shows weighted contributions,
base score, final score, and any penalty or cap adjustments.
So users can now verify how the score is formed, not just read the final number.

We also added a direct methodology flow and API health access from the header.
This makes it easier to understand the scoring model and quickly validate backend status during demos.

Next, we improved comparative analysis with token A versus token B mode,
so users can quickly see which token appears safer and why.

On reliability and QA, we added a dedicated Week 3 smoke suite that validates
top tokens, score, score breakdown, compare, search, history, and profile endpoints
in one repeatable command.

Week 3 turns TrustLayer from a working prototype into a transparent and demo-ready risk product.
Next, Week 4 will focus on hardening and reliability tuning under provider constraints.

4) Editing Notes
==================================================

- Use clean cuts, no heavy transitions.
- Prefer short captions focused on one feature per shot.
- If an adjustment row is not visible in Score Math for current token, narrate that
  adjustments appear only when penalties/caps are applied.
- Export 1080p, 30fps.
- Keep final duration between 60 and 75 seconds.
