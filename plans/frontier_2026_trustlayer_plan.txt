==================================================
1) CILJ I STRATEGIJA
==================================================

Primarni cilj:
- Uci u nagradjenu grupu i dobiti interview za Colosseum Accelerator.

Sekundarni cilj:
- Imati "investable" MVP (radi live, ima korisnicki problem, ima biznis pravac).

Strategija pobede:
- Jedan use-case uradjen vrhunski (token risk score), bez sirenja na 10 feature-a.
- Strong demo + cist GitHub + jasan business narativ.
- Fokus na kvalitet i pouzdanost, ne na broj ekrana.

==================================================
2) STA TACNO GRADIMO (MVP SCOPE)
==================================================

MVP funkcije (obavezno):
1. Search tokena po mint adresi.
2. Risk score 0-100 + status:
   - Green (nizi rizik)
   - Yellow (srednji rizik)
   - Red (visi rizik)
3. Objasnjenje score-a kroz 5 glavnih signala:
   - Holder concentration (koliko supply drzi top wallet-i)
   - Liquidity signal (da li postoji i kolika je likvidnost)
   - Mint/freeze authority signal
   - Token metadata verifikacija
   - Basic activity signal (trades/holders trend)
4. History panel (promena score-a kroz vreme).
5. Public API endpoint:
   - GET /v1/score/:mint

Post-MVP / stretch (samo ako ostane vremena):
- Wallet extension warning widget.
- Alerting (Telegram/Discord webhook kada score padne).
- Community review layer (user flag + evidence link).

3) PREPORUCENI TECH STACK
==================================================

Frontend:
- Next.js 14+ (TypeScript)
- Tailwind CSS
- Solana Wallet Adapter (za wallet connect ako zatreba)

Backend/API:
- Node.js 20 + Fastify ili NestJS
- TypeScript
- Zod za validaciju inputa

Data + storage:
- PostgreSQL (Supabase/Neon)
- Redis (cache score rezultata)

Solana/data integracije:
- @solana/web3.js
- Helius RPC / Enhanced API (ili drugi kvalitetan Solana provider)
- Birdeye/Jupiter podaci za market signale (ako API uslovi dozvoljavaju)

DevOps:
- Monorepo (pnpm workspaces ili turbo)
- GitHub Actions (lint + typecheck + test)
- Deploy:
  - Frontend: Vercel
  - API + worker: Railway/Fly/Render

Observability:
- Sentry (error tracking)
- Simple structured logs

4) ARHITEKTURA (MINIMALNA ALI OZBILJNA)
==================================================

Komponente:
1. Web app (UI)
2. API servis
3. Scoring engine (pravila + ponderi)
4. Background worker (periodicno osvezavanje score-ova)
5. PostgreSQL + Redis

Flow:
- User unese mint -> API validira -> proveri cache -> ako nema:
  - povuce onchain/offchain signale
  - izracuna score
  - snimi rezultat
  - vrati UI-ju score + reasons
